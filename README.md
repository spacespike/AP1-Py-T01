# 🐍 Project: AP1-Py-T01 — Core Algorithmic Engineering

An intensive, 10-assignment computational track with focus on algorithmic complexity, raw mathematical data parsing, input stream synchronization, and data lifecycle handling using **Python 3.x**.

---

## 🚀 Blueprint

This project moves away from third-party libraries and framework dependencies to emphasize low-level data structures, algorithmic memory complexity ($O(N)$ execution boundaries), and raw input stream manipulation.

* **Execution Runtime:** Python 3.1x (Core Builtins)
* **Design Standards:** Strict PEP 8 guidelines / Google Python Style Guide compliance
* **Verification Layers:** Advanced custom exception mappings and type validation workflows

---

## 📅 Roadmap

The solution for each task is encapsulated within its own file:


| Script File | Core Domain | Data Streams | Technical Mechanism |
| :--- | :--- | :--- | :--- |
| **`task1.py`** | 3D Spatial Vector Analysis | `stdin` $\rightarrow$ `stdout` | Matrix transformations and real-number dot products. |
| **`task2.py`** | Non-String Palindromes | `stdin` $\rightarrow$ `stdout` | Mathematical inversion using base-10 numerical truncation. |
| **`task3.py`** | Computer Vision Topology | `input.txt` $\rightarrow$ `stdout` | 8-connected component analysis & shape classification. |
| **`task4.py`** | Pascal's Triangle Gen | `stdin` $\rightarrow$ `stdout` | Combinatorial binomial sequencing with runtime type checks. |
| **`task5.py`** | Custom Float Interpreter | `stdin` $\rightarrow$ `stdout` | Lower-level char-by-char math evaluation (banning `float()`). |
| **`task6.py`** | Sorted JSON Record Merge | `input.txt` $\rightarrow$ `stdout` | Stable $O(N+M)$ double-pointer array merging. |
| **`task7.py`** | Optimal Path Token Collector | `stdin` $\rightarrow$ `stdout` | 2D Dynamic Programming (DP) route optimization. |
| **`task8.py`** | Matrix Item Deduplication | `stdin` $\rightarrow$ `stdout` | Hash-bucket lookup logic for set memory extraction. |
| **`task9.py`** | Polynomial Derivative Engine | `stdin` $\rightarrow$ `stdout` | Power-rule calculus calculations over variable exponents. |
| **`task10.py`** | Multi-Variable Constraint Match | `stdin` $\rightarrow$ `stdout` | Grouped hash mapping for global cost-minimization pairs. |

---

## 💡 Highlights

* **Algorithmic Logic:** Designed multi-dimensional grid traversals (Task 3) and predictive dynamic path routing matrices (Task 7) without importing advanced data science frameworks.
* **Low-Level Datatype Parsing:** Engineered parsers capable of unpacking floating-point structures directly from string layouts using character bit logic and decimal multipliers.
